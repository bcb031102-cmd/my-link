# 마이링크 (MyLink) - 상세 사용자 시나리오

본 문서는 마이링크 서비스의 소유자(Owner)와 방문자(Visitor)가 겪게 될 주요 흐름과 예외 상황을 시나리오 형태로 정의합니다.

---

## 1. 소유자 관점 (Owner Scenario)

### 1.1 최초 가입 및 초기 설정
- **로그인**: 사용자는 자신의 링크 페이지를 개설하기 위해 **구글 소셜 로그인 버튼을 클릭**한다.
- **초기 슬러그 설정**: 사용자는 자신의 지메일 아이디를 기반으로 한 **고유 URL(displayName)을 자동으로 할당받기 위해** 최초 로그인을 완료한다.
- **슬러그 중복 대응**: 사용자는 시스템으로부터 **"이미 사용 중인 이름입니다"라는 알림을 확인하기 위해** 자신의 지메일 아이디가 중복된 상황에서 다른 `displayName`을 입력한다.
- **로그인 성공 후 이동**: 사용자는 자신의 페이지를 바로 편집하기 위해 **로그인 직후 편집 모드가 활성화된 본인의 공개 페이지로 이동**한다.

### 1.2 링크 및 프로필 관리
- **링크 추가 (자동 제목)**: 사용자는 새로운 링크를 쉽고 빠르게 등록하기 위해 **URL만 입력하여 자동으로 웹사이트 제목(Title)이 채워지는 것을 확인**한다.
- **링크 최신순 정렬**: 사용자는 가장 중요한 최신 정보를 상단에 노출하기 위해 **링크를 등록하고 최신순으로 자동 정렬되는 것을 확인**한다.
- **인라인 수정**: 사용자는 복잡한 모달 창 없이 정보를 변경하기 위해 **페이지 내 텍스트(제목, URL, 이름 등)를 직접 클릭하여 인라인으로 수정**한다.
- **링크 삭제**: 사용자는 더 이상 필요 없는 링크를 제거하기 위해 **삭제 버튼을 클릭하여 즉각적인 반영을 확인**한다.

### 1.3 공개 및 로그아웃
- **본인 페이지 공유**: 사용자는 자신의 마이링크를 타인에게 공유하기 위해 **상단의 '내 페이지 링크 복사' 버튼을 클릭**한다.
- **로그아웃**: 사용자는 계정 보안을 유지하기 위해 **로그아웃 버튼을 클릭하여 메인 페이지로 이동**한다.

---

## 2. 방문자 관점 (Visitor Scenario)

### 2.1 페이지 탐색 및 상호작용
- **페이지 방문**: 방문자는 소유자의 정보를 확인하기 위해 **`mylink.io/displayName` 주소로 접속**한다.
- **링크 클릭 (카운트 증가)**: 방문자는 관심 있는 링크로 이동하기 위해 **버튼을 클릭하며, 이때 시스템은 소유자의 데이터베이스 내 해당 링크의 클릭 횟수를 1 증가**시킨다.
- **파비콘 확인**: 방문자는 링크의 신뢰도를 시각적으로 확인하기 위해 **버튼 왼쪽에 자동으로 표시된 사이트 파비콘(Favicon)을 참고**한다.
- **서비스 가입 유도**: 방문자는 자신만의 링크 페이지를 갖고 싶어 하기 위해 **페이지 최하단의 '나도 만들기' 버튼을 클릭**한다.

---

## 3. 예외 케이스 (Exception Cases)

### 3.1 인증 및 접근 오류
- **인증 실패**: 사용자는 로그인 과정에서 문제가 발생할 경우 **"로그인에 실패했습니다"라는 에러 메시지를 확인하기 위해** 인증 팝업을 닫거나 취소한다.
- **잘못된 페이지 접근**: 방문자는 존재하지 않는 주소로 접속할 경우 **"페이지를 찾을 수 없습니다"라는 404 안내를 확인하기 위해** 오타가 포함된 URL을 입력한다.

### 3.2 입력 유효성 관련
- **잘못된 URL 입력**: 사용자는 유효하지 않은 형식의 주소를 등록하지 않기 위해 **형식에 어긋난 URL 입력 시 나타나는 경고 표시를 확인**한다.
- **중복 슬러그 변경**: 사용자는 중복된 `displayName`을 사용하지 않기 위해 **변경 완료 전까지는 저장 버튼이 비활성화되거나 경고가 지속되는 것을 확인**한다.
- **네트워크 단절**: 사용자는 데이터 저장 실패를 인지하기 위해 **네트워크가 끊긴 상태에서 수정을 시도하고 나타나는 재시도 안내를 확인**한다.
