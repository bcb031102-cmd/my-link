# 마이링크 (MyLink) - PRD (기능 정의서)

## 1. 프로젝트 개요
### 1.1 프로젝트명
- **마이링크 (MyLink)**

### 1.2 목적
- 개인의 주요 링크를 모아 제공하는 아주 심플한 **'디지털 명함'** 서비스입니다.
- 불필요한 기능(이미지 업로드, 테마 등)을 배제하고 직관적인 편집과 정보 전달에 집중합니다.

### 1.3 대상 사용자
- 복잡한 설정 없이 빠르게 자신만의 링크 페이지를 만들고 싶은 모든 사용자

### 1.4 기술 스택
- **Frontend**: Next.js (App Router), React, Tailwind CSS
- **UI library**: **shadcn/ui**
- **Authentication**: **Firebase Auth (Google Social Login)**
- **Database**: **Cloud Firestore**
- **Favicon API**: **Google S2 Favicon API**

## 3. 기능 상세 설명

### 3.1 인라인 편집 및 정렬
- **UX**: 모든 정보는 별도의 페이지 이동 없이 해당 위치에서 즉시 수정할 수 있는 인라인 방식을 채택합니다.
- **정렬**: 링크는 등록된 순서 중 **최신순(Desc)**으로 상단에 배치됩니다.
- **자동화**: URL 입력 시 백엔드 또는 API를 통해 사이트 제목을 자동으로 추출하여 제목 필드를 채워줍니다.

### 3.2 이름 및 고유 주소 체계 (중복 체크 포함)
- **displayName (URL Slug)**: 
  - 시스템 내 고유값이며, 유일하지 않을 경우 사용자에게 **"이미 사용 중인 이름입니다. 다른 이름을 입력해 주세요."**라는 알림을 표시합니다.
  - **초기 설정**: 구글 로그인 시 지메일 아이디의 앞부분을 기본값으로 할당하되, 중복 시 사용자에게 수정 단계를 거치게 합니다.
- **username (Displayed Name)**: 프로필 페이지 상단에서 방문자에게 노출되는 사용자의 실제 별칭입니다.

### 3.3 사용자 공개 페이지
- **클릭 이벤트**: 방문자가 링크 버튼을 클릭하면, 외부 페이지로 이동하기 직전에 해당 링크의 `clicks` 필드 값을 1 증가시킵니다.
- **공통 요소**: 페이지 상단의 '내 링크 복사' 버튼과 최하단의 '내 페이지 만들기' 버튼을 항상 노출합니다.

## 4. 데이터 저장 구조 (Firestore)

### 4.1 데이터 스키마
- **users** {collection}
  - **[uid]** {document}
    - **displayName**: string (**URL 슬러그**, 고유값, 필수)
    - **username**: string (**프로필 노출 이름**, 필수)
    - **bio**: string
    - **email**: string
    - **links** {sub-collection}
      - **[linkId]** {document}
        - title: string
        - url: string
        - **clicks**: number (기본값 0)
        - createdAt: timestamp

## 5. UI/UX 디자인 원칙
1. **Zero Asset Upload**: 이미지 업로드 기능을 배제하여 사용성을 높입니다.
2. **Instant Feedback**: 중복 아이디 체크나 URL 제목 추출 시 즉각적인 시각적 피드백을 제공합니다.
3. **Data-Driven**: 소유자가 자신의 링크가 얼마나 효과적인지 '클릭수'를 통해 바로 체감할 수 있게 합니다.
